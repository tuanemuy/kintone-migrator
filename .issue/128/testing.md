# 動作確認計画 — Issue #128: initのスペース未指定対応

**Issue:** #128
**作成日:** 2026-03-26

---

## 確認環境

- ローカル開発環境
- kintone の開発環境（ドメイン、APIトークンまたはユーザー認証が必要）
- スペースに属するアプリとスペースに属さないアプリの両方が存在する kintone 環境が望ましい

## 確認項目

### 1. スペース未指定で全アプリ取得

- **目的:** `--space-id` を指定せずに `init` コマンドを実行し、すべてのアプリが取得されること
- **手順:**
  1. `pnpm dev -- init --domain {domain} --api-token {token}` を実行（`--space-id` なし）
  2. spinner が "Fetching all apps..." と表示されること
  3. "Found N app(s)." と表示されること
  4. 生成された設定ファイルにスペースに属さないアプリも含まれていること
- **期待結果:** 認証ユーザーがアクセス可能な全アプリが取得され、プロジェクト設定が正常に生成される
- **確認ポイント:** アプリ数が kintone 管理画面と一致するか

### 2. スペース指定時の既存動作

- **目的:** `--space-id` を指定した場合の既存動作が維持されていること
- **手順:**
  1. `pnpm dev -- init --space-id {spaceId} --domain {domain} --api-token {token}` を実行
  2. spinner が "Fetching space info..." と表示されること
  3. "Found N app(s) in the space." と表示されること
- **期待結果:** 指定したスペースに属するアプリのみが取得される（従来と同じ動作）
- **確認ポイント:** 従来の動作が一切変わっていないこと

### 3. dry-run での動作確認

- **目的:** `--dry-run` オプションとの組み合わせが正常に動作すること
- **手順:**
  1. `pnpm dev -- init --domain {domain} --api-token {token} --dry-run` を実行
  2. ファイルが実際に書き込まれず、プレビューが表示されること
- **期待結果:** dry-run 時に取得したアプリ一覧のプレビューが表示され、ファイルは生成されない

## エッジケース・異常系

### 1. アプリが0件の場合

- **目的:** アプリが1件もない場合にエラーが適切に表示されること
- **手順:**
  1. アプリが存在しない（またはアクセス権限がない）APIトークンで `pnpm dev -- init --domain {domain} --api-token {token}` を実行
- **期待結果:** 適切なエラーメッセージが表示される（"No apps found. Please check your API token permissions." 等）

### 2. 無効な `--space-id` の指定

- **目的:** 無効なスペースIDが指定された場合のバリデーションが動作すること
- **手順:**
  1. `pnpm dev -- init --space-id abc --domain {domain} --api-token {token}` を実行
  2. `pnpm dev -- init --space-id 0 --domain {domain} --api-token {token}` を実行
  3. `pnpm dev -- init --space-id -1 --domain {domain} --api-token {token}` を実行
- **期待結果:** 正の整数チェックでバリデーションエラーが表示される

### 3. 大量アプリの環境

- **目的:** 100件以上のアプリがある環境でページネーションが正しく動作すること
- **手順:**
  1. 100件以上のアプリが存在する kintone 環境で `pnpm dev -- init --domain {domain} --api-token {token}` を実行
- **期待結果:** 全アプリが取得される（100件を超えるアプリも含む）

## 既存機能への影響確認

- `--space-id` を指定した既存フローが一切変更されていないこと
- 生成される設定ファイルのフォーマットがスペース指定/未指定で同一であること
- `--help` 出力で `--space-id` がオプショナルパラメータとして表示されること
