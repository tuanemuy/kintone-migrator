# アプリケーションサービスの実装を行う

## 背景

- `spec/index.md` に設計を行った
- `docs/backend_implementation_example.md` にコアアーキテクチャの実装例を記載した

## タスク

1. `spec/usecases.tsv` を参照し、未実装のユースケースを確認する
2. 設計の実装に必要な部分を確認する
    - `spec/usecases/${domain}.md` の各ユースケースの詳細
    - `spec/domains/${domain}.md` のドメインモデル
2. 設計に従ってユースケースの実装を続ける
3. 型チェックとリンターを実行し、問題がなくなるまで修正を続ける
4. 実装が完了したユースケースについて、 `spec/usecases.tsv` の実装ステータスを更新する

## 条件

- ドメインロジックをアプリケーションサービス内に実装しない
- 入力には適切なバリデーションや存在チェックを行う
- DTOを活用して適切な型を出力する
- その他DDDの原則に反しない
- 設計に問題がある場合は質問する
